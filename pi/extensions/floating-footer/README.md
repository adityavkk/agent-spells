# floating-footer

Installable alternative to `minimal-footer`.

Features:
- opencode-inspired darker footer band
- subtle top divider and padded footer rows
- profile-aware model resolution for synthetic `profiles/<profile:role>` models
- footer shows both active profile status and concrete provider/model
- quota bars follow the resolved concrete provider, so Codex bars still show under model-profiles

Usage:
- enable `floating-footer`
- disable `minimal-footer` if you only want one custom footer active

Path:
- `~/dev/agent-spells/pi/extensions/floating-footer/`
